# 产品需求文档（PRD）- FitVenture 校园运动冒险

**文档版本：** MVP v1.3（School MVP / PRD-Demo 对齐版，术语与数值修复）  
**v1.3 修订摘要：** ① 删除「轻度疾风过载」，统一使用状态表中的「疾风过载」；② 锁定 Demo 初始进化倾向 wind 85 / iron 10 / hybrid 5（疾风过载），训练后 wind 78 / iron 16 / hybrid 6（疾风偏向），数值与生态状态完全对齐；③ Twin Form 命名统一为「疾风跑者 / 重装核心 / 混合运动者 / 恢复光环 / 连胜光环」，不再使用「疾风-重装混合型」等非表内名称；④ 玩家伙伴术语统一为「主宠」；⑤ 角色页快捷按钮锁定为三个：开始有氧、记录力量、查看隐藏线索；⑥ MVP 演示版以 Columbia 哥大校园为参考，地标使用真实建筑名（健身房 / 图书馆 / 篮球场 等），通用类型与具体地标互为同义词。  
**产品名称：** FitVenture  
**当前阶段定位：** 校园 LBS 运动冒险产品  
**MVP 范围：** 仅覆盖单一学校校园地图，不做城市级开放地图  
**最终提交方式：** GitHub Repository + GitHub Pages 交互式 HTML Demo；README 作为项目展示页和宣传页。  
**核心一句话：** FitVenture 将真实校园变成可探索的运动冒险地图，用户通过跑步、健身、恢复等真实运动行为捕捉宠物、训练宠物、推动宠物进化；宠物会被用户的运动习惯、校园路线、社交协作和行为序列共同塑形，同时 AI Body Twin 会根据个人运动习惯实时成长为更有风格的运动分身。

## 一、产品概述与核心价值（Product Vision）

### 1.1 核心痛点

大学生并不是完全不想运动，而是经常缺少持续运动的动机。传统运动 App 更像工具，主要记录步数、距离、心率、卡路里，但对用户来说即时反馈不够强，也缺少“我为什么今天还要继续练”的游戏化驱动力。

FitVenture 想解决的核心问题包括：

- **运动缺少即时正反馈：** 跑步、健身的真实身体变化周期较长，用户很难在短期内感受到成就。
- **传统健身 App 工具属性太强：** 数据记录本身不够有趣，很难长期留存。
- **校园运动社交弱：** 大学生在同一校园内有很高的物理密度，但缺少低压力、轻量化的运动社交方式。
- **过度训练风险：** 如果只用排行榜和数值刺激用户，容易导致为了刷分而过度运动。
- **普通 Avatar 系统缺少自我投射：** 传统捏脸或皮肤系统和用户真实运动行为关系弱，容易变成装饰品。

### 1.2 核心解决方案

FitVenture 以真实运动数据作为游戏资源，把校园生活路线转化为冒险路线。用户在校园中移动、跑步、健身、恢复，都会转化为游戏内的成长资源。

产品核心不是单纯“健身版 Pokémon Go”，而是：

> **校园 LBS 探索 + 运动习惯塑形的宠物进化 + AI Body Twin 自我成长 + 阵营 / 组队重社交。**

其中：

- **宠物系统是主线：** 用户探索校园、捕捉宠物、训练宠物、推动宠物基础进化与隐藏形态解锁。
- **隐藏形态是第三大核心玩法：** 宠物高级分支由地点、时间、社交协作和行为序列触发，不是普通彩蛋。
- **AI Body Twin 是副线：** 用户的运动习惯会生成一个越来越有风格、越来越帅的运动分身。
- **真实运动是输入：** 有氧、无氧、恢复分别转化为不同资源。
- **校园地图是场景：** 地标、宿舍、学院、操场、健身房构成早期 MVP 的核心地图。
- **阵营与组队是社交层：** 阵营负责学校内的大社群叙事，组队负责好友 / 临时小队协作。

### 1.3 当前 MVP 为什么先做学校

MVP 阶段只做单一学校地图，不做城市级开放地图。

原因是：

- **校园边界清楚：** 学校天然有操场、健身房、宿舍、图书馆、教学楼、食堂等高频地标，适合设计 LBS 点位。
- **用户密度高：** 同一个校园内用户集中，冷启动比城市级地图更容易。
- **身份感强：** 学院、专业、宿舍、年级都可以形成阵营和轻社交关系。
- **运动场景真实：** 学生本来就会在校园中走路、去 gym、去操场跑步，不需要强行制造行为。
- **MVP 可控：** 单校地图可以降低地图制作、地标配置、运营活动和作弊风控的复杂度。

长期来看，FitVenture 可以从单一学校扩展到多个高校，再扩展到大学城和城市运动地图。但当前 PRD 的 MVP 范围只聚焦学校。

## 二、产品主线与核心玩法（Core Game Loop）

### 2.1 总体主线

FitVenture 的主线不是单纯收集宠物，也不是单纯记录运动，而是形成一个完整闭环：

> **现实校园探索 → 遭遇并捕捉宠物 → 通过真实运动训练宠物 → 宠物根据运动习惯塑形进化 → 发现隐藏形态线索 → 参与阵营 / 小队地标挑战 → 获得更强资源与身份展示。**

同时，用户的每一次运动也会被 AI Body Twin 读取，形成另一条自我成长反馈：

> **真实运动 → AI Body Twin 运动人格变化 → 解锁外观、动作、装备和 Buff → 反哺宠物探索与训练。**

### 2.2 三大核心玩法支柱

| 核心玩法 | 产品作用 | 用户动机 |
|---|---|---|
| 校园运动冒险 | 把学校地标变成可探索、可捕捉、可挑战的地图 | “我想看看校园里今天出现了什么。” |
| 宠物习惯塑形进化 | 宠物不是统一喂养升级，而是被用户运动习惯塑形 | “我的宠物为什么会长成这个样子？” |
| 隐藏形态探索 | 由地点、时间、社交和行为序列触发高级进化分支 | “我想解锁别人没有的隐藏形态。” |

### 2.3 宠物是主线，AI Body Twin 是自我投射

当前版本不建议只做角色进化，也不建议让角色进化抢走宠物主线。推荐结构是：

| 系统 | 产品作用 | 用户动机 |
|---|---|---|
| 宠物系统 | 捕捉、养成、进化、图鉴、隐藏形态、地标挑战 | “我想收集并培养更稀有、更独特的宠物。” |
| AI Body Twin | 展示用户自己的运动风格、气场、装备和训练状态 | “我想让我的运动分身越来越帅、越来越像我。” |
| 校园地图 | 提供探索路线、地标任务和社交场景 | “我想在真实校园中完成冒险。” |
| 运动系统 | 将有氧、无氧、恢复转化为游戏资源 | “我每次运动都会产生具体价值。” |
| 阵营 / 组队系统 | 提供大社群身份与小队协作 | “我想和朋友 / 宿舍 / 学院一起解锁更高级内容。” |

### 2.4 统一运动资源与用途

FitVenture 的核心差异化在于：不同类型的运动会转化为不同游戏资源，而不是所有运动都简单换成经验值。最终统一术语如下：

| 资源 | 来源 | 用途 |
|---|---|---|
| 热量能量 | 有氧、校园移动 | 训练宠物、扩大探索收益、孵化宠物蛋 |
| 力量核心 | 无氧训练 | 合成捕捉球、强化重装方向、触发部分隐藏线索 |
| 捕捉球 | 力量核心合成 / 地标奖励 / 新手任务 | 捕捉宠物 |
| 恢复值 | 拉伸、轻走、休息日 | 降低疲劳、保护收益、辅助部分隐藏形态 |
| 坚持值 | 连续打卡 | 解锁称号、动作、外观光环 |
| 阵营贡献值 | 参与阵营任务、地标挑战 | 学院 / 宿舍排行、地标占领 |
| 羁绊值 | 宠物陪伴运动 | 解锁互动、进化前置条件、共鸣任务 |
| 隐藏线索 | 地点、时间、社交、行为序列 | 解锁隐藏形态 |

### 2.5 典型用户路径

#### Day 1：第一次进入校园地图

用户输入基础信息和运动目标，系统生成初始 AI Body Twin。用户打开校园地图，看到操场、健身房、图书馆、宿舍区等地标被轻度迷雾覆盖。

用户在校园中步行或慢跑 15 分钟后，地图被逐步点亮，并在操场附近遭遇第一只校园宠物。用户消耗新手捕捉球完成捕捉。

#### Day 3：开始训练宠物

用户进行一次 30 分钟有氧跑步，获得热量能量。系统提示可以用热量能量训练宠物，提高宠物等级或羁绊值。

同一天，用户的 AI Body Twin 出现“疾风跑者”视觉反馈，例如鞋底风轨、轻甲纹路、移动残影。

#### Day 5：无氧训练产生捕捉资源

用户去学校健身房完成一次上肢力量训练，获得力量核心和捕捉球。AI Body Twin 的肩背区域出现力量纹路，站姿更稳定，装备更偏重装。

用户之后在图书馆附近捕捉一只稀有宠物，因为力量核心提供了捕捉成功率加成。

#### Day 7：发现隐藏形态线索

系统根据一周数据总结用户的运动人格。如果用户有氧偏多，宠物可能偏向疾风形态；如果无氧偏多，宠物可能偏向重装形态；如果有氧和无氧长期均衡，宠物可能偏向混合形态。

如果用户在特定地标、时间或社交场景下完成任务，则可能获得隐藏线索，例如“晨曦形态 1/3”或“熔炉形态 1/3”。用户的宠物获得对应 Buff，并可以参加周末校园地标挑战。

## 三、MVP 目标与边界（MVP Scope）

### 3.1 MVP 核心目标

MVP 的核心目标是验证以下闭环是否成立：

> **用户是否愿意为了捕捉和训练宠物而进行更多真实运动？**  
> **用户是否会因为 AI Body Twin 的视觉成长而产生更强自我驱动力？**  
> **隐藏形态是否能提高探索欲、社交传播和长期留存？**  
> **阵营和小队机制是否能把单人运动转化为校园社交行为？**

### 3.2 MVP 当前只做学校版本

MVP 阶段只覆盖一个学校。推荐选择一个地理边界清晰、学生密度高、运动设施集中、校园身份感强的学校作为试点。

**MVP 演示版使用通用校园建筑名**，避免与任何具体高校绑定。Demo 中所用 5 个核心地标如下：

| 地标 | 主要资源 / 玩法定位 |
|---|---|
| 足球场 | 热量能量、疾风宠物、晨曦隐藏信号、跑步任务 |
| 健身房 | 力量核心、捕捉球、重装宠物、熔炉隐藏信号 |
| 图书馆 | 稀有宠物蛋、专注任务、夜间隐藏信号 |
| 宿舍区 | 每日补给、阵营贡献、宿舍协作事件 |
| 篮球场 | 个人路线、混合训练、投篮挑战 |

可扩展地标（V2 阶段）：教学楼 A / 教学楼 B、食堂 / 咖啡厅、校门 / 标志性建筑、中央广场 / 草坪。

阵营命名采用通用学院结构（如「工学院阵营」「文学院阵营」「宿舍 5 号楼」等），避免使用任何具体高校的院系缩写。README 中应注明：本项目为通用校园产品原型，与任何具体高校无商业合作关系。

### 3.3 MVP 不做的事

为了保证开发和验证成本可控，MVP 阶段暂不做以下内容：

- 不做城市级开放地图；
- 不做多个学校之间的跨校系统；
- 不做复杂实时 PVP 战斗；
- 不做自由宠物交易；
- 不做复杂即时聊天系统；
- 不接入自研 IoT 手环；
- 不做过度精细的身体扫描或医疗级健康判断；
- 不把角色外观与体重、体脂直接强绑定；
- 不鼓励用户为了刷分进行高风险运动。

## 四、核心功能需求（P0/P1）

## 4.1 校园 LBS 探索与宠物捕捉系统 - P0

### 功能描述

用户在真实校园中移动，App 根据 GPS 位置展示手绘风格校园地图。地图上分布不同资源点、宠物遭遇点、隐藏信号和地标挑战点。用户可以在校园中探索、开雾、遭遇宠物并进行捕捉。

### 核心机制

#### 1. 地图开雾

用户首次进入时，校园地图部分区域被迷雾覆盖。用户通过真实移动点亮地图，增强探索感。

规则示例：

- 步行或跑步经过某区域后，该区域被点亮；
- 高频运动路线会逐渐形成个人“运动轨迹”；
- 已探索区域可以出现更高等级宠物、隐藏线索或特殊事件。

#### 2. 地标资源点

校园内不同地点对应不同资源：

| 地点 | 资源 / 事件 |
|---|---|
| 操场 | 热量能量、疾风宠物、晨光隐藏信号、跑步任务 |
| 健身房 | 力量核心、捕捉球、重装宠物、熔炉隐藏信号 |
| 图书馆 | 稀有宠物蛋、专注任务、夜间隐藏信号 |
| 宿舍区 | 每日补给、阵营贡献、宿舍协作事件 |
| 食堂 / 咖啡厅 | 恢复资源、宠物羁绊任务 |
| 校园广场 | 公共活动、限时挑战、阵营展示 |

#### 3. 遭遇与捕捉

用户在特定区域移动时，有概率遇到宠物。捕捉需要消耗捕捉球，捕捉球主要通过力量核心合成、无氧训练、每日任务或地标补给获得。

捕捉成功率受到以下因素影响：

- 捕捉球等级；
- 宠物稀有度；
- 用户 AI Body Twin 当前形态；
- 是否完成对应地标任务；
- 用户疲劳状态；
- 当前宠物生态是否严重失衡。

#### 4. 宠物图鉴

用户捕捉到的宠物进入校园图鉴。不同学校后续可以拥有不同校园限定宠物，但 MVP 阶段只做单校图鉴。

图鉴内容包括：

- 已捕捉宠物；
- 未发现宠物剪影；
- 基础形态收集进度；
- 隐藏形态收集进度；
- 主宠设置；
- 宠物羁绊状态。

### MVP 验收标准

- 用户可以在校园地图上看到自己位置；
- 用户可以通过移动点亮地图；
- 用户可以在不同地标遭遇宠物；
- 用户可以消耗资源捕捉宠物；
- 捕捉后的宠物可以进入图鉴并展示在角色页；
- 用户能看到隐藏形态剪影，并能将一只宠物设为主宠。

## 4.2 宠物训练、习惯塑形与基础进化系统 - P0

### 功能描述

宠物系统是 FitVenture 的主线成长系统。用户捕捉宠物后，需要通过真实运动获得资源来训练宠物，最终推动宠物升级、基础进化和隐藏形态解锁。

### 核心设计原则

宠物不是被统一喂养升级，而是被玩家的真实运动习惯、校园路线、社交关系和训练节奏共同塑形。

宠物成长不能只靠在线时长，也不能只靠氪金，而是主要由真实运动行为驱动。

### 宠物基础形态

宠物基础形态统一为三类：

| 宠物形态 | 主要来源 | 设计说明 |
|---|---|---|
| 疾风形态 | 有氧占比较高，但仍需基础无氧 | 轻盈、速度感、探索能力强 |
| 重装形态 | 无氧占比较高，但仍需基础有氧 | 稳定、力量感、捕捉和挑战能力强 |
| 混合形态 | 有氧和无氧长期均衡 | 综合能力强，更容易触发高级隐藏线索 |

恢复 / 拉伸 / 休息日不作为宠物主进化路线，但会作为科学运动正反馈，用于降低疲劳、保护收益，并辅助部分隐藏形态触发。

### 训练资源

| 资源 | 来源 | 用途 |
|---|---|---|
| 热量能量 | 跑步、步行、骑行等有氧运动 | 宠物训练、升级、孵蛋 |
| 力量核心 | 健身房无氧训练 | 合成捕捉球、强化重装方向、解锁部分隐藏线索 |
| 捕捉球 | 力量核心合成、地标奖励 | 捕捉宠物 |
| 羁绊值 | 连续携带宠物运动 | 解锁动作、表情、进化剧情、共鸣任务 |
| 恢复值 / 恢复 Buff | 拉伸、休息日、轻运动 | 降低训练消耗，提高次日收益，辅助隐藏形态 |

### 宠物进化条件

宠物进化建议采用“多条件触发”，避免单纯刷步数：

- 达到一定等级；
- 完成一定次数真实有氧训练；
- 拥有指定数量力量核心；
- 达到指定羁绊值；
- 用户疲劳值不过高；
- 完成一个校园地标任务；
- 当前宠物生态不处于严重失衡状态。

示例：

> 一只速度型宠物从初阶进化到疾风形态，需要用户累计完成 10km 校园跑步、携带该宠物完成 3 次操场任务，同时至少完成 1 次基础无氧训练，避免纯有氧偏科。

### 宠物与 AI Body Twin 的联动

宠物不是孤立成长，AI Body Twin 会影响宠物训练效率。

| AI Body Twin 状态 | 对宠物系统的加成 |
|---|---|
| 疾风跑者 | 探索范围增加，疾风宠物训练效率提升 |
| 重装核心 | 捕捉球产出提高，重装宠物捕捉率提升 |
| 混合运动者 | 宠物综合训练效率提升，疲劳惩罚降低 |
| 恢复光环 | 恢复效率提升，宠物训练冷却缩短 |
| 连胜光环 | 连续打卡期间，宠物羁绊值获得额外加成 |

### MVP 验收标准

- 用户可以用有氧产生的资源训练宠物；
- 用户可以用无氧产生的资源捕捉或强化宠物；
- 至少存在一条可完成的宠物进化链路；
- 宠物成长和用户真实运动行为有明确关联；
- 宠物页能展示最近 14 天运动结构和进化倾向。

## 4.3 隐藏形态系统 - P0

### 功能描述

隐藏形态是 FitVenture 的第三大核心玩法，不是普通彩蛋。它是由玩家的校园地点、运动时间、社交协作和行为序列共同触发的个性化宠物进化分支。

隐藏形态服务于三个目标：

1. 增强探索欲；
2. 增强社交传播；
3. 增强长期留存。

### 四类隐藏形态触发条件

| 类型 | 示例 | 作用 |
|---|---|---|
| 地点隐藏形态 | 操场、健身房、图书馆、宿舍区 | 强化校园 LBS 玩法 |
| 时间隐藏形态 | 清晨、夜晚、考试周、开学周 | 制造仪式感和活动窗口 |
| 社交隐藏形态 | 好友协作、阵营任务、宿舍挑战 | 强化重社交属性 |
| 行为序列隐藏形态 | 先有氧后无氧、长期平衡、连续路线 | 强化个性化和策略性 |

### MVP 展示的 4 个隐藏形态

| 隐藏形态 | 类型 | 线索文案 | 下一步引导 |
|---|---|---|---|
| 晨曦形态 | 时间 + 地点 | 它似乎会在第一缕晨光照到操场时苏醒。 | 明早 6:00-8:00 携带疾风偏向宠物完成一次操场轻跑。 |
| 熔炉形态 | 健身房 + 无氧 + 基础有氧 | 它同时聆听铁器的节奏和脚步的回声。 | 在 健身房 完成力量训练后，再完成一次基础有氧来稳定路径。 |
| 双生羁绊形态 | 社交协作 | 它无法独自觉醒。 | 邀请同一名好友完成 3 次共鸣运动。 |
| 混合巅峰形态 | 长期平衡 | 当疾风与重装长期保持平衡时，它会接近真正的完整形态。 | 连续 2 周保持有氧 / 无氧平衡，并参与一次阵营任务。 |

### 设计原则

- 隐藏条件不完全公开，只给线索；
- 线索必须有明确下一步建议；
- 隐藏形态不能只靠氪金或随机获得；
- 高级隐藏形态应鼓励社交协作和运动平衡；
- 恢复行为可以作为辅助条件，但不作为主进化方向；
- 每个隐藏形态都应和校园场景或社交关系绑定，避免变成普通皮肤。

### MVP 验收标准

- 宠物页有“隐藏形态档案”入口；
- 至少展示 4 个隐藏形态剪影；
- 每个隐藏形态至少有 1 条线索；
- 用户完成一次演示流程后，可以解锁“熔炉形态 1/3”线索；
- 至少一个隐藏形态需要好友协作或阵营任务。

## 4.4 宠物生态平衡系统 - P0

### 功能描述

宠物生态平衡用于解决玩家只做有氧或只做无氧的问题。它不采用硬性上限，而是通过收益变化引导玩家保持基础运动平衡。

核心逻辑：

> 玩家可以偏向某一种训练风格，但如果整个主力宠物队伍过度单一，就会出现生态失衡，影响探索、捕捉、地标挑战和隐藏形态触发。

### 状态定义

| 状态 | 条件示例 | 影响 | 推荐行动 |
|---|---|---|---|
| 平衡 | 无单一形态超过 66% | 正常收益，隐藏线索更容易出现 | 继续保持 |
| 疾风偏向 | 疾风占比 67%–83% | 探索小加成，无明显惩罚 | 可补充力量训练 |
| 疾风过载 | 疾风占比 ≥84% | 捕捉材料产出下降，力量型挑战收益下降 | 完成力量训练或培养重装宠物 |
| 重装偏向 | 重装占比 67%–83% | 捕捉小加成，无明显惩罚 | 可补充有氧探索 |
| 重装饱和 | 重装占比 ≥84% | 探索范围下降，野外遭遇率下降 | 完成有氧训练或培养疾风宠物 |
| 混合平衡 | 混合形态占比提升且有氧无氧均衡 | 地标挑战适配度提升，隐藏线索增加 | 参与团队挑战 |

### 产品价值

- **科学运动：** 鼓励有氧和无氧都做；
- **游戏策略：** 队伍配置有取舍；
- **社交协作：** 玩家需要找训练风格互补的朋友；
- **个性化：** 不同用户会形成不同宠物生态。

### MVP 验收标准

- 角色页和宠物页都能看到当前生态状态；
- Demo 初始状态可以设置为“疾风过载”（即疾风形态占比 ≥84%）；
- 用户完成一次力量训练后，生态状态应向平衡方向变化；
- 生态状态会影响隐藏线索或捕捉资源产出提示。

## 4.5 AI Body Twin：个人运动分身系统 - P0

### 功能描述

AI Body Twin 是玩家在 FitVenture 世界中的运动分身。它不是普通捏脸系统，也不是单纯健康数据面板，而是根据用户真实运动行为实时成长的“运动人格体”。

AI Body Twin 不应简单根据体重、体脂、BMI 来评判用户外观，而应根据用户的训练类型、训练频率、训练平衡度、恢复情况和长期坚持度，生成不同的视觉风格、动作姿态、装备组件和运动 Buff。

### AI 到底在哪里

MVP 阶段的 AI 可以先用规则 + 生成式表现模拟，但 PRD 中必须明确 AI 能力方向：

1. 根据用户训练结构、疲劳值、坚持值和宠物生态生成个性化运动分身视觉；
2. 根据用户近期状态生成身体天气和训练建议；
3. 根据行为模式推荐可能的隐藏形态线索；
4. 后续版本可接入生成式视觉模型，生成更高定制化的 Avatar 形象。

这样避免评审认为 AI Body Twin 只是规则映射。

### 核心原则

#### 1. 角色要帅，但帅不等于瘦或壮

角色的帅来自训练风格、气场、动作、装备和坚持感，而不是只来自身材变化。

错误方向：

> 体重下降 → 角色变好看；体脂下降 → 角色更高级。

推荐方向：

> 跑步多 → 角色变得轻盈、有风轨、有速度感。  
> 无氧多 → 角色装备更厚重、力量纹路更明显。  
> 训练均衡 → 角色形态更完整，气场更稳定。  
> 恢复做得好 → 角色出现恢复光环和稳定状态。

#### 2. 角色成长要短期、中期、长期都有反馈

| 反馈周期 | 用户行为 | AI Body Twin 反馈 |
|---|---|---|
| 即时反馈 | 完成一次运动 | 出现对应能量粒子、动作变化、局部光效 |
| 周期反馈 | 一周训练结构形成 | 生成本周运动人格形态 |
| 长期反馈 | 连续数周坚持 | 解锁高级装备、称号、姿态、主题皮肤 |

#### 3. 角色不直接变成战斗力，而是提供风格与 Buff

AI Body Twin 不应该变成“谁练得狠谁战斗力高”的系统，而应该是身份表达和轻量加成系统。

推荐属性：

| 属性 | 来源 | 作用 |
|---|---|---|
| Stamina 耐力 | 有氧运动 | 增加探索半径、提升陪跑宠物收益 |
| Power 力量 | 无氧训练 | 增加捕捉球产出、提升捕捉成功率 |
| Balance 平衡 | 有氧、无氧、恢复结构均衡 | 降低疲劳惩罚，提高综合收益 |
| Recovery 恢复 | 休息、拉伸、轻运动 | 提升次日训练收益 |
| Consistency 坚持 | 连续打卡 | 解锁外观、称号、宠物羁绊值加成 |

### Twin Form：每周运动人格形态

系统每周根据用户训练结构生成一个 Twin Form。最终定义如下：

| 状态 | 类型 | 视觉表现 | 游戏加成 |
|---|---|---|---|
| 疾风跑者 | 训练风格形态 | 风轨、轻甲、披风、速度残影 | 探索范围 +10%，疾风宠物训练加成 |
| 重装核心 | 训练风格形态 | 肩背护甲、金属纹路、拳套、力量站姿 | 捕捉球产出 +15%，捕捉成功率小幅提升 |
| 混合运动者 | 训练风格形态 | 完整套装、稳定能量核心、均衡体态 | 宠物训练效率 +10%，疲劳惩罚降低 |
| 恢复光环 | 辅助状态 | 治愈光环、稳定光效、舒展动作 | 次日恢复收益 +20%，训练冷却缩短 |
| 连胜光环 | 叠加状态 | 连续光环、专属称号、特殊入场动作 | 宠物羁绊值额外加成 |

说明：恢复光环不是宠物主进化路线，也不是独立的宠物基础形态，而是 AI Body Twin 的辅助状态，用于保留“恢复也是正反馈”的产品哲学。

### MVP 验收标准

- 用户初次进入可以生成基础 AI Body Twin；
- 用户完成不同类型运动后，AI Body Twin 有不同即时视觉反馈；
- 系统可以根据一周训练结构生成至少 3 种训练风格形态；
- 恢复光环和连胜光环可作为叠加状态展示；
- AI Body Twin 的形态至少影响一个宠物捕捉或训练参数；
- 角色外观不与体重/体脂直接强绑定，避免身材焦虑。

## 4.6 科学运动、疲劳值与恢复系统 - P1

### 功能描述

FitVenture 需要避免用户为了刷资源而过度训练。因此 MVP 中需要加入基础疲劳机制，让科学运动成为产品差异化，而不是只做游戏刺激。

### 疲劳值与身体天气的关系

不能用身体天气替代疲劳值。最终定义：

- **疲劳值：** 明确的运动科学状态，数值化展示；
- **身体天气：** 由疲劳值、训练结构、坚持状态生成的游戏化氛围表达。

示例：

```text
疲劳值：68 / 100
身体天气：疾风偏向，轻雾
```

### 疲劳值机制

用户每次运动会产生疲劳值。不同运动影响不同：

- 有氧运动增加全身疲劳；
- 无氧训练增加对应部位疲劳；
- 高强度连续训练会提高疲劳积累速度；
- 休息、拉伸、低强度活动会降低疲劳。

### 收益递减

当疲劳值过高时：

- 运动资源收益下降；
- 宠物训练效率下降；
- AI Body Twin 出现疲劳视觉反馈；
- 系统推荐恢复任务。

### 恢复奖励

恢复不是宠物主进化方向，但必须是科学运动正反馈。

恢复 / 拉伸 / 休息日保留三个价值：

1. 降低疲劳；
2. 保护训练收益；
3. 辅助部分隐藏形态触发。

示例：

- 完成一次 10 分钟拉伸，获得恢复值；
- 设置一天为休息日，次日训练收益提升；
- 疲劳值恢复到健康区间后，AI Body Twin 获得更稳定的恢复光环；
- 宠物训练冷却缩短。

### MVP 验收标准

- 用户可以看到基础疲劳值；
- 疲劳过高时收益会递减；
- 恢复行为可以带来正向 Buff；
- 身体天气可以作为疲劳值的游戏化表达，但不能替代疲劳值；
- 系统不会鼓励用户无限刷运动量。

## 4.7 阵营、小队与重社交系统 - P1

### 功能描述

校园场景最大的优势是身份感和密度。FitVenture 的社交不应只是排行榜，而要同时包含：

- **阵营：** 学院 / 宿舍 / 年级等大社群身份；
- **组队：** 好友或临时玩家组成的小队协作。

二者叠加，不互相替代。

### 阵营划分

MVP 阶段可以按以下方式之一划分阵营：

- 学院 / School；
- 专业 / Major；
- 宿舍楼 / Dorm；
- 年级 / Class Year。

推荐 MVP 优先使用宿舍或学院，因为身份感强，也容易形成对抗。

### 地标挑战

每周统计某个校园地标附近的运动贡献值。贡献最高的个人或阵营获得地标展示权。

展示方式：

- 该地标上方出现获胜玩家的 AI Body Twin；
- 展示主宠；
- 展示本周称号，例如 “Gym Guardian” 或 “Library Spirit Keeper”；
- 其他用户经过该地标时可以看到。

### 小队协作与共鸣进化

高级隐藏形态和地标 Boss 需要小队协作。

示例：

- 双生羁绊形态需要同一好友完成 3 次共鸣运动；
- 地标 Boss 需要至少 1 只疾风宠物、1 只重装宠物、1 只混合宠物；
- 宿舍挑战可以通过多人贡献热量能量、力量核心和阵营贡献值完成。

### 阵营页结构

```text
阵营页
├── 我的阵营
├── 阵营排行榜
├── 地标 Boss
├── 推荐小队
├── 好友互补推荐
└── 共鸣进化任务
```

### MVP 验收标准

- 用户可以加入一个阵营；
- 阵营有周贡献进度；
- 地标可以展示获胜玩家或获胜阵营；
- 用户可以看到其他人的 AI Body Twin 和主宠；
- 阵营页同时展示大社群阵营和小队协作任务；
- 至少一个隐藏形态需要好友协作或小队任务。

## 五、MVP 数据采集、反作弊与数值骨架

## 5.1 MVP 运动数据采集方案

### 有氧

MVP 方案：App 内模拟 / 记录 GPS 跑步与步行，也可先用手动模拟展示核心闭环。

真实版本可接入：

- Apple Health；
- Google Fit；
- Strava；
- 手机 GPS。

### 无氧

MVP 方案：手动记录训练部位 + 时长。

字段：

- 训练部位；
- 训练时长；
- 主观强度；
- 地点是否为校园健身房。

真实版本可扩展：

- 健身房扫码；
- 设备传感器；
- 校园健身房预约 / 打卡数据。

### 恢复

MVP 方案：手动记录拉伸、轻走、休息日。

### Demo 处理方式

HTML Demo 不接入真实 GPS 和健康数据，只用前端 mock 状态模拟以下路径：

> 初始疾风过载（wind 85 / iron 10 / hybrid 5）→ 点击 健身房 → 记录 35 分钟力量训练 → 获得力量核心和捕捉球 → 生态从“疾风过载”缓解为“疾风偏向”（wind 78 / iron 16 / hybrid 6）→ 解锁熔炉形态 1/3 线索 → 宠物进化倾向更新。

## 5.2 基础反作弊策略

MVP 不需要完美反作弊，但需要说明风控思路：

1. GPS 路线合理性检测；
2. 速度区间检测；
3. 异常重复轨迹标记；
4. 单日收益上限；
5. 疲劳收益递减；
6. 阵营榜按参与率和生态平衡加权；
7. 无氧训练采用时长上限和部位疲劳限制；
8. 地标贡献不展示实时精确位置，降低代刷和尾随风险。

## 5.3 数值平衡骨架

### 产出

| 行为 | 产出 |
|---|---|
| 1km 有氧 | 热量能量 +80 |
| 20 分钟有氧 | 热量能量 +160，羁绊值 +8 |
| 20 分钟无氧 | 力量核心 +2，疲劳值 +6 |
| 35 分钟无氧 | 力量核心 +4，捕捉球 +2，疲劳值 +10 |
| 10 分钟拉伸 | 恢复值 +15，疲劳值 -12 |
| 完成一次阵营地标任务 | 阵营贡献值 +50，隐藏线索概率提升 |

### 消耗

| 行为 | 消耗 |
|---|---|
| 捕捉普通宠物 | 捕捉球 -1 |
| 捕捉稀有宠物 | 捕捉球 -2，力量核心 -1 |
| 训练宠物一次 | 热量能量 -120 |
| 进化预览 | 热量能量 -200，力量核心 -2 |
| 解锁隐藏形态最终阶段 | 根据形态消耗热量能量、力量核心、羁绊值或阵营贡献值 |

## 六、页面结构设计（HTML Demo / App Demo）

MVP 推荐采用四个底部 Tab，最终命名统一为：

```text
角色｜探索｜宠物｜阵营
```

不要把“宠物”改成“进化”，因为图鉴、主宠设置、羁绊、隐藏形态都属于宠物系统。不要把“阵营”改成“组队”，因为阵营是更大的校园社交叙事，组队是阵营页内部的小队协作功能。

### 6.1 【角色】Tab：AI Body Twin 首页

核心作用：让用户看到“自己正在变强”。

页面内容：

- AI Body Twin 大图展示；
- 当前 Twin Form / 运动分身状态；
- 今日身体天气；
- 明确的疲劳值；
- 当前资源：热量能量、力量核心、捕捉球、恢复值、坚持值；
- 当前宠物生态状态；
- 主宠同框展示；

## 七、冷启动、空状态与回流设计

### 7.1 Day 1 新用户

新用户没有宠物和好友时：

- 完成新手校园探索；
- 获得第一只校园幼兽；
- 自动加入宿舍 / 学院阵营；
- 推荐 1 个轻量任务；
- 展示“今日只需 10 分钟校园探索即可完成首日目标”。

### 7.2 没有好友

- 阵营页显示推荐队友；
- 可先参加公共阵营任务；
- 不强迫邀请通讯录好友；
- 社交隐藏形态显示“可稍后解锁”，不阻断主线进度。

### 7.3 没有宠物

- 宠物页显示新手引导：前往探索页捕捉第一只校园幼兽；
- 图鉴展示剪影和首只宠物获得路径；
- 角色页的主宠位置显示“等待你的第一个伙伴”。

### 7.4 断签回流

不要让断签惩罚过重。

推荐文案：

```text
欢迎回来。你的宠物进入休眠状态，但不会消失。完成一次轻量任务即可重新激活进化路径。
```

## 八、隐私与合规

FitVenture 涉及学生地理位置、运动数据、健康相关数据、校园社交关系，以及可能的未成年人数据。因此 MVP 阶段必须遵循以下原则：

1. **最小化采集：** 只采集完成核心闭环必要的数据；
2. **用户可删除数据：** 用户应能够删除运动记录、位置记录和账号数据；
3. **默认隐藏精确位置：** 社交展示中不公开用户实时精确位置；
4. **阵营排行榜不展示敏感健康数据：** 排行只展示贡献值，不展示体重、体脂等敏感指标；
5. **未成年人额外保护：** 若扩展到未成年人校园，需要额外家长同意与隐私保护；
6. **位置展示延迟或模糊化：** 地标贡献可以显示在地标层面，不暴露实时轨迹；
7. **AI Body Twin 避免身材羞辱：** 不使用“胖、瘦、丑、懒”等身体评价文案。

## 十、差异化与竞争壁垒

### 10.1 与 Pokémon Go 的区别

FitVenture 可以借鉴 LBS 探索和捕捉的直觉，但核心逻辑不应停留在“现实地图抓宠物”。

| Pokémon Go | FitVenture |
|---|---|
| 现实地图探索是核心入口 | 校园地图探索是运动场景入口 |
| 主要目标是抓宝可梦和收集 | 主要目标是通过运动训练并进化宠物 |
| 走路主要用于孵蛋、找点、拿补给 | 有氧、无氧、恢复分别对应不同成长资源 |
| 宠物进化路径较固定 | 宠物会被真实运动习惯、校园路线和社交关系塑形 |
| 玩家自身角色成长较弱 | AI Body Twin 是个人运动分身和自我投射 |
| 地图是全球大众地图 | MVP 聚焦校园高密度社区 |
| 社交以团战和道馆为主 | 社交以校园阵营、小队共鸣、地标挑战和身份展示为主 |

FitVenture 的差异化可以总结为：

> Pokémon Go 奖励玩家到处走；FitVenture 奖励玩家建立更持续、更科学、更有身份感的运动习惯。

### 10.2 核心壁垒

#### 1. 校园场景壁垒

校园不是随便换一张地图，而是一个高密度、强身份、强地标、强重复行为的真实场景。FitVenture 可以围绕学校地标、学院阵营、宿舍文化设计内容。

#### 2. 运动资源系统壁垒

不是简单把步数换经验，而是把有氧、无氧、恢复拆成不同资源，形成更科学的运动闭环。

#### 3. AI Body Twin 壁垒

AI Body Twin 将用户真实运动行为转化为视觉身份，而不是普通 Avatar 皮肤。它既能满足“变帅”的情绪价值，也避免直接制造身材焦虑。

#### 4. 宠物养成与真实运动绑定

宠物成长必须来自真实运动行为，形成比普通收集游戏更强的现实行为驱动。

#### 5. 隐藏形态与社交传播壁垒

隐藏形态不是静态皮肤，而是由地点、时间、社交和行为序列共同触发。它天然鼓励玩家讨论线索、邀请朋友和反复探索校园。

## 十二、未来扩展路线（Roadmap）

虽然当前 MVP 只做学校版本，但长期不应把产品永远锁死在单一校园。推荐路线如下：

### V1：单校 MVP

目标：验证核心闭环。

范围：

- 一个学校；
- 校园地图；
- 宠物捕捉与训练；
- 隐藏形态；
- AI Body Twin；
- 基础阵营与地标挑战。

### V2：多校版本

目标：验证校园网络效应。

新增：

- 多学校地图；
- 每个学校的限定宠物；
- 跨校访问徽章；
- Inter-Campus Challenge；
- 学校之间的运动排行榜。

示例：

- Columbia：狮子系、城市系、图书馆系宠物；
- MIT：机器人系、科技系、实验室系宠物；
- NYU：城市街区系宠物；
- UCLA：阳光、海岸、运动系宠物。

### V3：大学城地图

目标：把多个校园连接成区域网络。

例如：

- Boston / Cambridge 大学圈；
- NYC 大学校园网络；
- LA 大学校园网络。

新增：

- 大学城跑步路线；
- 跨校地标任务；
- 城市运动事件；
- 品牌联名挑战。

### V4：城市运动冒险地图

目标：扩展到更大众的城市级运动生活方式平台。

新增：

- 公园跑步任务；
- 城市步道；
- 商业健身房合作；
- 马拉松 / 运动节活动；
- 城市限定宠物。

## 十三、商业化与生态愿景（V2.0+）

MVP 阶段不以商业化为核心，但可以预留后续方向。

### 13.1 虚拟外观与主题皮肤

用户可以通过长期运动、活动挑战或付费解锁：

- AI Body Twin 外观；
- 宠物皮肤；
- 全局 UI 主题；
- 入场动作；
- 地标展示特效。

需要注意：付费内容不应破坏运动公平性，优先做外观，不做强数值付费。

### 13.2 校园联名活动

可以与学校健身房、学生社团、运动品牌合作：

- 校园 7-Day Fitness Quest；
- 学院运动挑战；
- 新生周校园探索活动；
- 健身房打卡活动。

### 13.3 实体周边

后续可以将高人气宠物和 AI Body Twin 元素转化为：

- NFC 徽章；
- 宠物盲盒；
- 校园限定贴纸；
- 运动装备联名。

### 13.4 IoT 运动设备接入

后续可以接入 Apple Health、Google Fit、Garmin、Strava 或自有手环，提高运动识别准确性。

MVP 阶段不做自研硬件，只预留接口。

## 十四、总结

FitVenture 的当前 MVP 应该聚焦学校，而不是一开始做全城市。校园具备高密度、强身份感、清晰地理边界和真实运动场景，是验证 LBS 运动冒险最合适的切入点。

产品主线应明确为：

> **宠物捕捉、习惯塑形进化与隐藏形态是主线；AI Body Twin 是玩家自我投射与运动人格成长副线；真实运动是驱动两者成长的核心资源；阵营和组队让运动从单人打卡变成校园社交。**

最终，用户的心理动机应该是：

> 我想让宠物进化，所以我开始运动；  
> 我运动之后发现自己的 AI Body Twin 也越来越帅、越来越有风格；  
> 我通过隐藏形态和阵营任务发现朋友也很重要；  
> 我因此更愿意持续运动，并在校园里形成身份感和社交参与感。
