## 0. 最重要的对齐原则

本 Demo 的目标不是做一个普通运动打卡 App，也不是做一个简单的 Pokémon Go 模仿品，而是展示 FitVenture 的 MVP 闭环：

> 用户在真实校园中探索、捕捉宠物，通过有氧和无氧运动获得不同资源；宠物会根据用户真实运动习惯、校园路线、社交关系和训练节奏发生不同进化；AI Body Twin / 运动分身则展示用户自身的训练风格和身体状态；隐藏形态和重社交协作构成长期留存动力。

当前 Demo 必须遵守：

1. **只做单校园 MVP**，不做城市级地图。
2. **所有用户可见文案使用中文**。
3. **四个底部 Tab 命名必须与 PRD 对齐**：`角色｜探索｜宠物｜阵营`。
4. “宠物页”内部可以重点展示“进化实验室”，但 Tab 名称仍然叫“宠物”。
5. “阵营页”内部可以包含好友组队功能，但 Tab 名称仍然叫“阵营”。
6. “隐藏形态”是第三大核心玩法，必须显著展示，不能只作为彩蛋。
7. “宠物生态平衡”是核心策略系统，必须在角色页和宠物页都能看到。
8. “恢复”不是主进化路线，但仍然是科学运动正反馈，用于疲劳管理、收益保护和隐藏条件辅助。
9. Demo 最终只要求使用 GitHub 提交：GitHub Pages 作为可交互 final product，GitHub README 作为项目展示页 / 宣传页。不要再要求 Netlify、Vercel 或 CodeSandbox。

## 1. 产品定位

FitVenture 是一款 **校园 LBS 运动冒险产品**。

它将真实的校园移动、有氧训练、无氧训练、恢复行为和校园社交转化为：

- 宠物捕捉；
- 宠物进化；
- 隐藏形态探索；
- AI Body Twin / 运动分身成长；
- 宠物生态平衡；
- 宿舍 / 学院 / 好友协作挑战。

核心体验：

> 为了宠物开始运动，为了隐藏形态持续探索，为了朋友和阵营一起运动，为了自己的运动分身变强而长期坚持。

### 5.3 宠物页

宠物页是核心玩法页，不要只叫“进化页”，但内部主模块可以叫“进化实验室”。

必须包含：

1. 当前宠物卡片；
2. 图鉴入口；
3. 最近 14 天训练结构；
4. 当前进化倾向；
5. 基础形态路径；
6. 隐藏形态档案馆；
7. 宠物生态平衡影响；
8. 羁绊值。

当前宠物示例：

```text
宠物：校园幼兽
等级：Lv.8
羁绊值：Lv.7
状态：正在被你的运动习惯塑形
```

最近 14 天训练结构：

```text
有氧：58%
无氧：32%
恢复：10%
```

注意：恢复不作为主进化路线，但用于降低疲劳、保护收益和辅助隐藏形态。

进化倾向（初始演示状态，对应「疾风过载」）：

```text
疾风形态：85%
重装形态：10%
混合形态：5%
```

基础形态卡：

- 疾风形态：由有氧主导，但需要基础力量达标；
- 重装形态：由无氧主导，但需要基础有氧达标；


点击图鉴可以打开弹窗或抽屉，展示：

- 已捕捉宠物；
- 未发现剪影；
- 已解锁隐藏形态；
- 主宠设置按钮。

## 6. AI Body Twin / 运动分身设计

“AI Body Twin”不能只是规则面板。Demo 中需要解释 AI 在哪里。

在 MVP Demo 中，AI Body Twin 的 AI 部分可以定义为：

1. 根据用户训练结构、疲劳值、坚持值、宠物生态状态生成个性化运动分身视觉；
2. 根据近期运动行为生成身体天气和训练建议；
3. 每周生成运动人格形态，如疾风跑者、重装核心、混合运动者、连胜光环；
4. 后续可扩展为生成式视觉 Avatar 和个性化推荐。

Demo 页面文案可写：

```text
AI Body Twin 会根据你的有氧、无氧、恢复、疲劳和坚持状态生成当前运动分身。它不是评价你的身材，而是把你的训练风格可视化。
```

Twin Form 在 Demo 中保留 4 类：

| 类型 | 是否主展示 | 说明 |
|---|---|---|
| 疾风跑者 | 是 | 有氧偏向，探索能力强 |
| 重装核心 | 是 | 无氧偏向，捕捉资源强 |
| 混合运动者 | 是 | 有氧无氧均衡，适配度高 |
| 连胜光环 | 是，作为叠加状态 | 连续打卡形成的光环 / 称号 |

恢复相关的 Vital Guardian 不作为主形态展示，改为：

```text
恢复状态 / 恢复光环
```

它不是主进化路线，但会影响疲劳、安全和次日收益。

## 7. 恢复与疲劳系统

Demo 必须显式显示疲劳值。

示例：

```text
疲劳值：68 / 100
状态：中等偏高
建议：今天完成一次力量训练后，建议做 10 分钟拉伸来降低疲劳。
```

规则：

- 疲劳值低于 40：训练收益正常；
- 疲劳值 40–75：正常，但提示注意恢复；
- 疲劳值高于 75：训练收益下降，推荐恢复任务；
- 恢复 / 拉伸不作为主进化路线；
- 恢复值可降低疲劳、保护进化收益、辅助隐藏形态条件。

恢复任务示例：

```text
完成 10 分钟拉伸：恢复值 +15，疲劳值 -12，次日训练收益 +10%。
```

## 8. 宠物生态平衡系统

宠物生态平衡是核心策略机制。Demo 必须展示。

目标：

> 不硬性禁止玩家偏向某种训练风格，但如果宠物队伍过度单一，就会影响部分收益，引导玩家保持基础有氧和无氧平衡。

状态表：

| 状态 | 条件示例 | 影响 | 推荐行动 |
|---|---|---|---|
| 平衡 | 无单一形态超过 66% | 正常收益，隐藏形态线索更容易出现 | 继续保持 |
| 疾风偏向 | 疾风占比 67%–83% | 探索小加成，无明显惩罚 | 可补充力量训练 |
| 疾风过载 | 疾风占比 ≥84% | 捕捉材料产出 -10%，力量型挑战收益下降 | 完成力量训练或培养重装宠物 |
| 重装偏向 | 重装占比 67%–83% | 捕捉小加成，无明显惩罚 | 可补充有氧探索 |
| 重装饱和 | 重装占比 ≥84% | 探索范围下降，野外遭遇率下降 | 完成有氧训练或培养疾风宠物 |
| 混合平衡 | 混合形态占比提升且有氧无氧均衡 | 地标挑战适配度提升，隐藏线索增加 | 参与团队挑战 |

Demo 初始状态：

```text
当前状态：疾风过载
影响：捕捉材料产出 -10%
建议：在 健身房 记录一次 35 分钟力量训练
```

## 9. 隐藏形态系统

隐藏形态是第三大核心玩法，必须高度可见。

它不是普通彩蛋，而是长期留存、社交传播和个性化探索的核心。

隐藏形态由四类条件触发：

1. 地点：特定校园地标；
2. 时间：清晨、夜晚、考试周等；
3. 社交：好友协作、阵营任务；
4. 行为序列：先有氧后无氧、长期平衡、连续路线等。

### 9.1 隐藏形态档案馆

宠物页必须有一个显眼模块：

```text
隐藏形态档案馆
```

展示 4 个隐藏形态：

#### 1. 晨曦形态

类型：时间 + 地点

线索：

```text
它似乎会在第一缕晨光照到操场时苏醒。
```

条件方向：

- 清晨；
- 操场；
- 疾风偏向宠物；
- 不能处于严重疾风过载。

进度：

```text
线索 1 / 3
下一步：明天 6:00–8:00 在操场完成一次轻跑。
```

#### 2. 熔炉形态

类型：地点 + 无氧 + 基础有氧

线索：

```text
它同时聆听铁器的节奏和脚步的回声。
```

条件方向：

- 健身房 力量训练；
- 最近 14 天有基础有氧；
- 不处于严重重装饱和。

进度：

```text
线索 1 / 3
下一步：完成一次有氧后，再到 健身房 记录力量训练。
```

#### 3. 双生羁绊形态

类型：社交

线索：

```text
它无法独自觉醒。
```

条件方向：

- 与同一好友完成 3 次协作训练；
- 至少一次阵营 / 地标任务；
- 好友宠物形态互补。

进度：

```text
协作训练 2 / 3
下一步：邀请 Jason 完成一次共鸣训练。
```

#### 4. 混合巅峰形态

类型：长期平衡

线索：

```text
当疾风与重装长期保持平衡时，它会接近真正的完整形态。
```

条件方向：

- 多周有氧无氧均衡；
- 宠物生态处于平衡或混合平衡；
- 参与至少一次阵营挑战。

进度：

```text
暂未解锁
下一步：让最近 14 天训练结构接近 50% 有氧 / 50% 无氧。
```

### 9.2 隐藏线索反馈

点击“发现隐藏信号”或完成训练后，需要弹出线索反馈：

```text
隐藏线索解锁
熔炉形态 1 / 3
你的校园幼兽对 健身房 的力量信号产生了反应。
下一步：完成一次有氧训练，让它的进化路径稳定下来。
```

## 10. 运动数据采集方案：MVP 模拟版

MVP Demo 必须给出清楚的数据采集假设。

正式产品可以后续接入 Apple Health、Google Fit、Strava、校园健身房扫码等，但 HTML Demo 阶段使用模拟数据。

### 10.1 有氧采集

MVP 方案：

- 用户点击“开始有氧”；
- Demo 模拟 GPS 路线和时长；
- 完成后生成热量能量、路线进度、宠物羁绊。

示例：

```text
模拟完成 2.4km 校园轻跑
热量能量 +220
羁绊值 +12
疲劳值 +8
个人灵脉经验 +20
```

### 10.2 无氧采集

MVP 方案：

- 用户手动记录训练部位和时长；
- 可选择胸 / 背 / 腿 / 肩 / 核心；
- 系统根据时长和部位生成力量核心、捕捉球、疲劳值。

示例：

```text
记录 35 分钟力量训练
训练部位：背部 + 核心
力量核心 +4
捕捉球 +2
疲劳值 +10
疾风过载降低
隐藏线索 +1
```

### 10.3 恢复采集

MVP 方案：

- 用户点击“记录恢复”；
- 选择拉伸 / 轻走 / 休息日；
- 获得恢复值并降低疲劳。

## 11. 反作弊策略：MVP 展示版

Demo 不需要真正实现反作弊，但需要在 UI 中展示基本策略。

在设置 / 信息弹窗中加入：

```text
MVP 反作弊策略：
1. 有氧运动需要 GPS 路线与速度区间合理；
2. 异常高速、原地抖动、重复轨迹会被标记；
3. 无氧训练采用手动记录 + 周期上限 + 疲劳收益递减；
4. 同一时间段不能重复获得多份运动收益；
5. 阵营榜采用参与人数和生态平衡加权，避免少数人刷榜。
```

Demo 中可用一句提示：

```text
当前为演示数据，真实版本将结合 GPS 合理性、设备传感器和收益上限进行基础反作弊。
```

## 14. 必须实现的交互故事线

## 19. 验收标准

Demo 完成后，用户或评审应该能在 60 秒内理解：

1. 这是一个校园 LBS 运动冒险产品；
2. 当前 MVP 只做学校版本；
3. 玩家通过有氧和无氧获得不同资源；
4. 宠物根据真实运动习惯进化，不是固定升级；
5. 隐藏形态是第三大核心玩法；
6. 阵营和好友组队是两层社交关系；
7. AI Body Twin / 运动分身展示玩家自己的训练风格；
8. 宠物生态平衡会引导用户不要只做有氧或只做无氧；
9. 恢复不是主进化路线，但仍然有正反馈；
10. Demo 四个 Tab 之间状态会同步。
